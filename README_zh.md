# WebDAV GPG Sync

> [English README](README.md)

> **⚠️ v2.0.0 破坏性变更 — 不向后兼容**
>
> v2.0.0 将加密方案从 **RSA 4096 (OpenPGP)** 切换为 **ML-KEM-768 (NIST FIPS 203 后量子加密)**。
>
> **旧的 GPG 密钥、passphrase 以及所有已加密的远程数据全部作废。** 更新后必须：
> 1. 在插件设置中重新生成密钥对（无需 passphrase）
> 2. 删除 WebDAV 服务器上的旧加密数据（或使用空目录重新开始）
>
> 密钥格式从 OpenPGP armor 改为 base64。设置项 `privateKey` 重命名为 `secretKey`。

**文件内容 + 文件名均加密。服务端只能看到随机数据。AES-256-GCM + ML-KEM-768 后量子端到端加密，支持任意 WebDAV。**

## 核心特性

### 🔒 全程加密，零信任云端

大多数同步方案（Obsidian Sync、iCloud、Dropbox 等）依赖服务端信任——服务商**可以**读取你的数据。**WebDAV PQC Sync 使用端到端加密。** 所有加密在本地完成，仅密文发送到 WebDAV：

- **✅ 文件名也加密** — 不仅仅是文件内容：**每个文件夹和文件名**都独立用 AES-256-GCM 加密并 Base64URL 编码。服务端只看到随机字符串，完全不知道你在编辑什么文件
- **文件内容加密** — **ML-KEM-768 (FIPS 203)** 密钥封装 + **AES-256-GCM** 批量加密。ML-KEM 输出的 32 字节共享密钥直接用作 AES-256-GCM 密钥，通过 Web Crypto API 获得硬件加速
- **后量子安全** — ML-KEM-768 是 NIST 标准化的 RSA/ECC 替代方案，能够抵抗量子计算机的 Shor 算法攻击
- **目录结构隐藏** — 完整文件夹层次映射为等深加密路径，云端无法重建你的文件夹布局
- **私钥永不离开本地** — 私钥仅存储在本地 Obsidian 配置中，上传加密只需公钥

**结果：WebDAV 服务商、网络中间人和任何有权访问服务器的人都只能看到加密后的二进制数据和随机文件名。零内容、零标题、零结构。**

### 🔄 双向同步

- **基于 ETag 的增量同步** — 仅传输变更文件
- **日志追踪** — 本地变更记录日志，崩溃后可恢复
- **自动冲突检测** — 当文件在本地和远端同时变更时，自动创建冲突副本（`.conflicted.YYYY-MM-DD.md`）
- 双向同步文件和目录的新建、修改、重命名和删除

### ⚡ 操作说明

| 操作 | 说明 |
|--------|------|
| **生成密钥对** | 一键生成 ML-KEM-768 后量子密钥对（无需 passphrase） |
| **同步到 WebDAV** | 将本地变更推送（加密后）到 WebDAV，同时拉取远端变更 |
| **从 WebDAV 恢复** | 在新设备上下载并解密 WebDAV 上的所有文件，重建整个仓库 |
| **自动同步** | 文件保存时自动同步（3 秒防抖） |
| **手动同步** | 功能栏按钮 + 命令面板 |

### 🛡️ 安全设计对比

| 项目 | v1.x（旧版） | v2.0+（当前） |
|-------|-----------|------------------|
| 密钥封装 | RSA 4096 (OpenPGP) | **ML-KEM-768 (NIST FIPS 203)** — 抗量子 |
| 批量加密 | AES-256-GCM | AES-256-GCM（不变） |
| 文件名加密 | AES-256-GCM（密钥来自 GPG 指纹） | AES-256-GCM（密钥来自 SHA-256(Kyber 公钥)） |
| 密钥派生 | PBKDF2 10 万次迭代 | PBKDF2 10 万次迭代（不变） |
| 密钥格式 | OpenPGP armor (ASCII) | **Base64 编码的原始字节** |
| 口令 (passphrase) | 必需（用于加密私钥） | **已移除**（密钥无加密存储） |
| 每文件加密开销 | 536 字节 | 1108 字节（增加 572 字节） |
| 私钥 → 公钥 | 可从私钥推导 | **必须同时存储两者**（Kyber 无法从私钥推导公钥） |
| 抗量子 | ❌ 否（RSA 可被 Shor 算法攻破） | **✅ 是** |
| 打包体积 | ~600 KB (OpenPGP.js) | **~72 KB** (@noble/post-quantum) |

### ⚙️ WebDAV 兼容性

适用所有标准 WebDAV 服务：

- NextCloud / ownCloud
- Synology NAS
- Apache mod_dav
- 任意支持 WebDAV 的云存储

支持自签名证书（在设置中启用）。

> **🌐 Cloudflare 代理兼容** — 上传和下载使用**分块传输**，自动拆分超过 100MB 的文件。这避免了 Cloudflare 的 100MB 上传限制，可以稳定工作在 Cloudflare 代理、CDN 及其他反向代理之后。上传和下载均支持分块。

### 🔁 分块传输

**超过 90MB**（可在高级设置中调整）的文件会自动拆分为多个块进行上传和下载：

- **上传**：使用 Nextcloud OC-Chunked 协议——逐个上传分块，然后在服务端组装
- **下载**：使用 HTTP `Range` 请求——**并行**获取分块（每块 10MB），每块超时 60 秒，自动重试 3 次
- **状态栏** 实时显示分块进度：`Uploading notes.md (3/12 · file 5/18)`
- 小于 5MB 的文件单次请求下载，速度最快

### 🚀 快速开始

1. 安装插件
2. 在设置中填写 WebDAV URL、用户名和密码
3. 点击 **Generate Key Pair** 生成 ML-KEM-768 密钥对
4. 点击 **Sync Now** 开始加密同步

> **迁移到新设备**：从旧设备导出配置（含密钥），在新设备上导入，然后使用 **Restore from WebDAV**。

### 排除路径

默认为 `.obsidian/plugins/` 和 `.trash/`。可按需添加更多逗号分隔的路径前缀。
